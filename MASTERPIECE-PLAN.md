## 3. חוויית-הקסם — התכנון

העיקרון המנחה: **הקסם חייב להיות (א) נראה לפני הרשמה, (ב) עקבי בתוך הספר, (ג) יוצא מהמסך אל העולם (קול → וידאו → נייר).**

### 3.1 עקביות-דמות (הדרישה המפורשת של אלעד) — "דמות אחת, כל העמודים"
הדפוס המוכח מ-`Documents/elad-brand-kit/illustration-styles/ELAD-CHARACTER.md` (דמות-אלעד העקבית) + ממצאי `RESEARCH-SELF-INSERT-FEATURE.md` (עקביות-דמות היא הממצא המרכזי ביכולות Gemini):

1. **Character Sheet פעם אחת:** בעת יצירת הספר מייצרים תחילה דיוקן-דמות אחד (avatar) לפי ה-Story Bible.
2. **Reference בכל עמוד:** כל איור-עמוד מקבל את תמונת-הדיוקן כ-reference + את תיאור-הדמות **מילה-במילה** מה-Bible (`characterPromptFor` כבר קיים) + עוגן-סגנון גלובלי אחד (`global_style`).
3. **שני מסלולי-מנוע:** (א) OpenAI images/edits — הצנרת קיימת (`referenceImageBase64`); (ב) Gemini `gemini-3-pro-image-preview` עם image-parts כרפרנס — עדיף כשעובדים על ה-free tier (דרישת אלעד: Gemini חינם). ה-endpoint `api/ai/gemini-image.js` קיים וכבר מטפל ב-gotcha של thinkingBudget; צריך להוסיף לו קבלת reference-image.
4. **בדיקת-עקביות אחרי יצירה:** צ'ק אוטומטי (אותה פלטת-צבע/תיאור; בעתיד — השוואת embedding) + כפתור "צייר מחדש" לעמוד בודד.
5. **החיווט חסר במקום אחד בלבד:** `BookWizard` (סביב שורות 885–920, לולאת יצירת התמונות) — לייצר avatar לפני הלולאה ולהעביר אותו לכל `generateImage`.

### 3.2 הקראה קולית — שלוש שכבות (לפי RESEARCH-TTS-VIDEO-FREEMIUM)
- **חינם:** Web Speech API (קיים היום בקורא).
- **רשום:** TTS ענן דרך `api/ai/tts.js` הקיים (gpt-4o-mini-tts עם פריסטים עבריים שכבר כתובים — bedtime/playful/dramatic-lite). לחבר לקורא + לקאש את ה-MP3 ב-Storage (עמוד שהוקרא פעם לא מיוצר שוב).
- **פרימיום (בהמשך):** ElevenLabs / Azure Hila+Avri — קול לכל דמות.
- חוויה: "הקראת לילה" — כפתור אחד בקורא: תאורה חמה + הקראה איטית + מוזיקה עדינה (הכל כבר קיים כרכיבים נפרדים — לחבר לפריסט אחד).

### 3.3 סרטוני-סיפור קצרים
- קיים: Remotion MVP שעובד ב-CLI (הפקת mp4 תקין, 7MB, אומת 25.5).
- מסלול רזה נכון-לעכשיו (בלי worker אסינכרוני): **הפקה בצד-שרת בעת בקשה מראש** — python/node + ffmpeg: תמונות-הספר + Ken-Burns + TTS שכבר קיים + מוזיקת-רקע חינמית = קליפ 30–60 שניות עם לוגו סיפוראי בסוף.
- הערך האמיתי: **לא פיצ'ר לילד אלא מנוע-שיווק** — ההורה משתף את הקליפ בוואטסאפ המשפחתי → כל שיתוף הוא פרסומת. לכן הקליפ חינמי ותמיד עם חתימת-מותג.

### 3.4 ספריית ז'אנרים ותבניות
קיים בסיס (ז'אנרים, סגנונות, רעיונות יומיים). השדרוג: 8–12 "תבניות-קסם" מוכנות בעמוד-הבית — "ספר יום-הולדת", "הרפתקה בגן", "ספר גמילה מחיתולים", "אח/ות חדש/ה", "ערב יום-כיפור לילדים" — כל אחת עם Story Bible חצי-מוכן. מקצר את הדרך לספר ראשון מ"דף ריק" ל-3 לחיצות, וגם נותן מדף-SEO (עמוד ציבורי לכל תבנית).

### 3.5 שמירה, שיתוף, הדפסה
- **שיתוף:** `books.is_public` + views מסוננים-PII כבר קיימים ועובדים. חסר: עמוד-שיתוף ציבורי יפה (OG-image של כריכת הספר) + כפתור-וואטסאפ. זה מנוע-הצמיחה המרכזי בשוק הישראלי.
- **הדפסה:** שלב 1 (ימים בודדים, אפס תלות): PDF עברי תקין ברזולוציית-דפוס + דף "איך מדפיסים" + לינק BookPod. שלב 2: אינטגרציית API מלאה (הזמנה מתוך האתר). שלב 3: כריכה קשה (Cloudprinter).
- **Self-Insert (תמונה→דמות):** ה-differentiator הגדול, אחרי שהבסיס חי. Tier-1 מה-MVP במחקר: העלאת תמונה בעמוד הדמויות → הסכמת-הורה → דמות מאוירת → שימוש כ-reference בכל ספר. פרטיות: "עבד-ומחק" (מקור נמחק תוך 5 דקות; נשמר רק האיור).

## 6. אופק ב' — המשך ליורש (אחרי שהבסיס חי)

**פאזה I — צמיחה אורגנית (שבוע 1–2):**
- עמוד-שיתוף ציבורי מלוטש (OG-image דינמי של הכריכה) + כפתור-וואטסאפ — מנוע-ההפצה המרכזי.
- 8–12 תבניות-קסם (§3.4) כעמודים ציבוריים (SEO/GEO — הפרוטוקול כבר על 97–100).
- מנוע-המלצות "עמוד מהספר" (הרעיון של אלעד ב-RESEARCH-PRINT): בקשת המלצה עם עמוד מאויר מהספר של הממליץ.
- מיזוג agent-API (הענף קיים) — קאמי/קיילי יוצרים סיפור בפקודה קולית של אלעד (דמו-שיווקי מעולה).

**פאזה II — מונטיזציה (שבוע 2–4):**
- "מהדורת מתנה" ₪29–39 (PDF-דפוס + הקראה + קליפ) — checkout חד-פעמי ב-Creem.
- שער-ביקוש להדפסה (§4) → אם עבר: BookPod שלב-2 (API מלא).
- קליפ-וידאו משותף (§3.3) — ffmpeg רזה, חינם, תמיד ממותג.

**פאזה III — בידול (חודש 2+):**
- Self-Insert Tier-1 עם "עבד-ומחק" (COPPA) — ה-moat האמיתי מול StoryJumper.
- קול-מותאם (ElevenLabs) כפיצ'ר פרימיום; מסלול-גננות B2B.
- וידאו-Remotion מלא עם worker אסינכרוני (הענף קיים) — רק אם הקליפ הרזה מוכיח ביקוש.

**קווים אדומים ליורש:**
- שום שינוי RLS/סכימה בלי preview + מועצה + גיבוי (הפרוטוקול הזה כבר הציל פעמיים).
- ילדים = רגולציה: כל פיצ'ר-תמונה עובר את מסך-הסכמת-ההורה; אפס PII ב-views ציבוריים (נשמר היום — לא לשבור).
- עלויות-תמונה: Gemini free-tier קודם; OpenAI רק כשיש reference-edit הכרחי; קאש אגרסיבי (איור שנוצר לא נוצר שוב).
- אסור לפרסם תוכן-ילדים בקהילה כברירת-מחדל — שיתוף הוא בחירה מפורשת של הורה.
